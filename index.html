2.1信息存储

1.字节（byte）是8位的块，作为最小的可寻址的存储器单位<而不是单独的位>.

2.存储器中每个字节由一个惟一的数字标识，这称之为他的地址。

3.所有可能地址的集合，称为虚拟地址空间。<像一个分好类别的仓库吧？>

4.编译和运行时，系统将存储空间划分为更可管理的单元，来存放不同程序对象（程序数据、指令和控制信息）。<分类存放，便于之后的管理与取用>

2.1.1十六进制表示法

因为二进制表示太冗长，而十进制与位模式的互相转化很麻烦，两种符号表示法都比较麻烦。所以采取16进制来书写位模式。编写机械级程序常常需要手工地进行十、二、十六之间的转换。需要掌握进制之间的转化方法。

2.2 整数表示

2.2.1整型数据类型

C支持多种整型数据类型——表示有限范围的整数。如char、short、int、、long。C标准定义了每种数据类型必须能够表示的最小值范围。具体而言，数据类型的取值会有差异。

2.2.2无符号和二进制补码编码

  • 二进制的补码范围是不对称的，容易产生错误。
  • 最大无符号值刚好比二进制补码的最大值的两倍大一（1）点。
  • 尽管C的标准没有要求用二进制补码形式来表示有符号整数，但几乎所有机器都是这么的。
  • 对有符号数，其最高有效位也称为符号位（sign bit），0为正，1为负。

2.2.3有符号数和无符号数之间的转换

依托两者之间的映射关系进行转换，具体方法如有需要：有符号数如何与无符号数转换 <用ctrl+点击>

2.2.4 C中的有符号与无符号数

C支持所有整数数据类型的有符号与无符号运算。尽管C标准没有指定某种有符号数的表示，但几乎所有机器都使用二进制补码。通常，大多数数字都是默认有符号的。（另外，当一种类型的表达式被赋值给另一种类型的变量时，转换是隐式发生的）

当执行一个运算时，如果它的一个运算数是有符号而另一个是无符号，那么C将有符号参数类型转换为无符号数，并假设两数非负，并执行。（于一些情况下会导致运算错误）

2.2.5扩展一个数字的位表示

一个常见的运算是在不同字长的整数之间转换，同时保持数值不变。而当目标类型太小了，则转换可能失效，但从较小到较大，应该总可能的。（*将一个无符号数转换为一个更大的数据类型，只要简单的在表示的开头添加0，这种运算被称为零扩展（zero extension）；要将一个二进制补码数字转换俄日一个更大的数据类型，规则是执行一个符号扩展（sign extension），在表示中添加最高有效位的值）。

从一个数据大小到另一个数据大小，以及无符号和有符号数字之间的转换的相对顺序能够影响一个程序的行为。

2.2.7有符号数和无符号数的建议

1.尽量不要在你的代码中使用无符号数，以免增加不必需要的复杂性。

2.尽量使用像int那样的有符号类型，这样在涉及升级混合类型的复杂细节时，不必担心边界情况。

3.只有在使用位段和二进制掩码时，才可以用无符号数。应该在表达式中使用强制类型转换，使操作数均为有符号或无符号数，这样就不必由编译器来选择结果的类型。

    3.尾数规格化：加减完成的尾数形式未必满足IEEE754对于尾数的要求，即保留23位，小数点在第一个1之后。需要对尾数进行左规和右规，下面介绍一下左规和右规。左规：将尾数向左移，用于清除第一个1前面出现的0。左规伴随着阶码减少，在左规过程中要检测阶码是否发生下溢，即阶码达到最小值（0000 0000）。右规：将尾数向右移，用于两数相加后出现向高位进位的情况。因为两数相加最多进一位，所以右规最多一位。右规伴随着阶码的增加，在右规过程中要检测阶码是否发生上溢，即阶码达到最大值（1111 1111）。

开始使用Linux操作系统

开始使用Linux操作系统

2-1；Linux 目录结构与访问路径

目录的基本概念

目录是描述文件夹的专有名词。

在计算机代码中，文件夹形式与书本目录相同。

  • 书本目录记录所有书的章节名和页码。
  • 计算机目录记录其中包含的文件和对应的访问地址。
  • 相对于普通文件，目录也称为目录文件。

Linux文件系统是树形结构

优点

  • 层次结构鲜明
  • 易于扩展分支
  • 分支隔离互不依赖

Windows系统每个盘符作为文件根节点（C盘。D盘）

Linux系统采用单根结构，整个文件系统只有一个根。

Linux的根目录（/）下有众多的二级目录，如

  • （/boot，存放Linux启动所需的内核文件和镜像文件）
  • (/bin,存放最常用的命令)
  • （/dev，存放Linux的外部设备，且在Linux中访问设备的方式和访问文件相同）
  • （/etc，存放所有系统管理所需要的配置文件和子目录）
  • （/home，用户的主目录，属于用户）
  • （/root，此目录为系统管理员，也称作超级管理员root用户的主目录）

访问路径

绝对路径（文件或目录相对于根目录的可访问路径）

形式特点：

  1. Windows：从盘符开始书写的路径。如：C：\Linux
  2. Linux：从根目录（/）开始书写的路径。如：/home

优缺：

  • 具有唯一性，但是只能在特定的操作系统环境下才能准确定位目标路径。

  • 不适合编写脚本。

相对路径（文件或目录相对于当前目录的可访问路径）

形式特点：

1.不是从根目录（/）开始书写的路径。如：etc。

2.一般从当前目录开始书写

tips：

用“.”表示本级目录；用“..”表示上级目录。

优缺：

  • 同一个参考体系内，相对路径不变，适合编写脚本。
  • 计算较为复杂，需要掌握文件系统的目录结构。

（归结而言一个是从根目录出发，一个是从其他的目录出发，但无论从什么地方出发，目的地是一样的，不过过程不同）

2-2定位和切换目录

定位当前目录（确定当前工作目录的绝对路径）

命令：pwd（打印当前工作目录）

输出：当前所在工作目录的绝对路径

（可用于相对路径的参考点）

切换工作目录（前往新的工作目录）

命令：cd（cd+【空格】+目标的绝对路径或相对路径）

tips：

{ ./ } 表示当前目录

{../}上一级目录

单走{cd}直接切换至用户家目录

{cd+空格+横线}切换到上一个工作目录（上一次工作的目录）

{cd+空格+波浪线}切换到用户家目录

2-3查看目录内容

Linux命令基本范式

含（命令关键词，命令选项，命令参数）

范式：命令关键字 选项 命令参数

其中：

命令关键字：必不可少，指明要完成的具体操作

选项：可选，指定用什么方式来完成操作，一般有默认值。

命令参数：可选，指明操作对象

查看目录内容

含（查看指定目标内容，按指定格式查看目录内容，按指定格式查看指定目录内容）

命令：ls 【选项】 【目录路径】（分别对应 ，命令关键字，选项，命令参数）

其中选项： 注：（可以有多个选项内容，如：ls -l -a / ； ls -al /）

  • l：长格式显示目录内容

  • a：显示目录所有内容，包括隐藏文件

  • 无选项：仅显示非隐藏文件的名称

其中目录路径：

  • 可以是绝对路径，也可为相对路径
  • 无参数：默认显示当前目录下的内容

tips：

{ll+空格+/}={ls+空格+-l+空格+/}

ll -d /dev（查看dev目录的相关信息）

ll /dev（查看dev目录的内容）（注意空格）

{ls+空格+-+-+help} 查看命令的使用格式与方法。

ll 查看当前目录内容

2-4创建目录和文件

新建目录，含(在指定位置新建目录，在指定路径上新建所有目录，同时新建多个目录)

命令：mkdir 【选项】 【目录路径】

其中目录路径：

  • 可以是绝对路径，也可以为相对路径
  • 可以是多个参数，表示一次性创建多个目录

其中选项：

  • p：若指定的路径上有不存在目录，一并创建‘
  • 无选项：仅创建指定路径上最后一个目标目录

tips：

mkdir 目录1 ；表示新建目录1

mkdir -p 目录2/目录3（此时会先在当前目录下创建目录2，再在目录2下创建目录3，，如果不使用-p，则无法执行命令）

新建文件，含(touch命令，echo命令输出重定向)

法1：touch 【文件路径】

其中文件路径：

  • 可以是相对路径也可以是绝对路径
  • 可以是多个参数，表示一次性创建多个文件

法2：vi 【文件路径】

其中文件路径：

  • 可以是绝对路径，也可以是相对路径
  • 可以是已经存在的文件（可以该编辑文件）
  • 也可以是还未创建的文件（新建并编辑该文件）

法3：echo 内容信息 >|>> 【文件路径】

输出重定向符：

  • {>}:将命令的结果输出从显示器改到文件（覆盖）
  • {>>}：将命令的结果输出从显示器改到文件（追加）

其中文件路径：

  • 可以是绝对路径，也可以是相对路径‘
  • 可以是已经存在的文件（编辑文件）

tips：

touch 文件1 |创建一个文件

touch 文件2 |文件3 ；创建多个文件

vi 文件1 |（因为已经存在，所以编辑文件）（会进入vi编辑器中，i键 进入插入模式 ，Esc键退回vi命令模式，输入 ： 符 ，进入vi底行模式 ，输入wq保存文件）

cat 文件1 |查看文件内容

cat 文件4 |文件不存在，则先创建再编辑

echo “这是文件5” > 文件5

echo “2002 3 2” >> 文件1（表示追加输入）
